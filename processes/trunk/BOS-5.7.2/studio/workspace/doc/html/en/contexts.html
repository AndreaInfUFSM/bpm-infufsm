Contexts

(Category: Contexts)

Contexts are useful to handle different “environments / status / contexts” of a process; for example, to test a process in a test environment before deploying it.

For example, by defining a “test context” and a “production context” and associating different variables to each context, there is no need to change values in connectors, forms, or other places where variables necessarily differ in different environments.

A context is a set of key-value pairings. Context keys (which will be replaced by a predefined value) can be used as input in most text variables.

When a process is associated with a context and then Run, the context keys are replaced by the associated values defined for that context.
