Define parameters

(Category: Parameters)

Parameters are used as placeholders for information in a process that is more appropriately defined outside the process. Parameter keys are defined inside a process, and values are defined outside the process. Those values are then assigned to their keys at runtime.

Parameter key-value pairs are useful to handle different environments of a process.

For example, by defining a local (test) environment and a production environment and associating the appropriate values to a key for each environment, there is no need to change values in actors, connectors, forms, or other places where values necessarily differ in different environments. For example, test and production may use different values for different specific users (beta testers vs end users) or for different server login and passwords.

Before you can run or export a process for deployment, all process parameters must be keyed to values.
