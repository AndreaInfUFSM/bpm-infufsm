Configure messages

(Category: Process Modeling)

Messages are used in a process to carry information from one process (pool) to another. Messages are unique; that is, a single message is sent and a target can receive a single message. This is useful to synchronize two processes, for example.

This is a definition specific to BPMN - these are not messages sent by email or chat. (To send email, see email connectors). Since transitions / process flow cannot cross process boundaries in BPMN, messages are used to carry information such as data from one process to another. This requires configuration called data mapping between the processes involved.

There five types of message events:

  • start message
  • throw message
  • catch message
  • boundary message
  • end message

There are two ways to add messages to a process design in Bonita Studio :

  • using drag and drop from the design palette
  • drag and drop via the context palette on an element
